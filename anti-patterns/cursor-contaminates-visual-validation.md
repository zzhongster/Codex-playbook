# 反模式：鼠标污染视觉验收夹具

## 一句话结论

依赖固定颜色、角标或像素占比的视觉验收，必须关闭鼠标渲染或显式排除鼠标区域；要求操作者“把鼠标移开”不是可重复验证。

## 场景

录屏探针在选区边缘绘制四种颜色标记，分析首帧来验证裁剪边界和方向。

## 失败表现

鼠标覆盖 12×12 角标后，颜色命中率低于阈值，系统报告 marker missing 或 orientation reversed。捕获坐标其实正确，验收却随机失败。

## 根因

鼠标是独立于静态夹具的动态像素源。只要启用 `showsCursor`，人工操作、鼠标停留位置和系统光标形态都会进入证据。

## 正确做法

优先让验证工作流显式关闭光标：

~~~swift
ScreenCaptureService(
    ...,
    showsCursor: false
)
~~~

共享服务的默认值可以保持 `true`，以保留正式录屏行为；只有探针覆盖默认值。若业务必须录鼠标，则验证器需要获取鼠标位置并屏蔽对应区域。

## 验证清单

- 配置对象有 `showsCursor` 行为测试。
- 正式录屏默认仍为 `true`。
- 探针源代码契约显式为 `false`。
- 最终实机证据在配置变更后重新采集。
- 不把“操作者没动鼠标”当独立可审计证据。
